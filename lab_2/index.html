Project Overview

Goal: Launch, configure, and connect to a Windows Server EC2 instance, demonstrate secure access, and run basic DOS commands. This page also covers recommended security group rules, key pair handling, and connection options.

Deliverable ✅ Launched EC2 Instance Deliverable ✅ Configured Security Group Deliverable ✅ Generated Key Pair Deliverable ✅ Successful Connection Deliverable ✅ Basic DOS Interaction
Heads‑up: Windows instances are typically accessed via RDP (port 3389). SSH is optional and requires enabling the OpenSSH Server feature on Windows.

Knowledge/Skills Prerequisites

  • Basic understanding of AWS EC2.
  • Familiarity with the Windows/DOS command line (Command Prompt or PowerShell).
  • Basic HTML/CSS (for static content like this guide).
  • Networking basics: security groups, ports, and IP addresses.

AWS Resources Used

  • AMI: Microsoft Windows Server (e.g., 2019/2022 Base)
  • Instance Type: t3.small (recommended) or t2.micro (testing)
  • VPC: Default VPC with a public subnet
  • Security Group: Inbound rules for RDP (3389) + optional HTTP/HTTPS
  • Key Pair: RSA (.pem or .ppk) to decrypt the Windows admin password
  • Elastic IP (optional): Stable public IP for production

Step‑by‑Step Deployment

  1. Launch Instance → EC2 Console > Launch instance.
    Name: Win-Server-EC2 • AMI: Windows Server 2019/2022 Base • Type: t3.small
    Network: Default VPC • Subnet: Public • Auto-assign Public IP: Enable.
  2. Create/Select Key Pair → Download .pem (Mac/Linux) or .ppk (PuTTY). Keep it private.
  3. Security Group → Add inbound rules (see section below). Launch the instance.
  4. Get Windows Password (after 4–5 minutes): EC2 > Instances > Select instance > Connect > RDP client > Get password. Upload your key to decrypt. Copy the Administrator password.
  5. Download RDP file from the same panel or open your RDP client. Use the instance’s Public IPv4 address.
  6. Connect as Administrator with the decrypted password. Accept the certificate prompt.
  7. Initial Config (optional): Run Windows Update, set time zone, and enable desired features (e.g., IIS).

Security Group — Recommended Rules

RDPTCP 3389Your IP (recommended)Remote Desktop access
HTTP (optional)TCP 800.0.0.0/0, ::/0Web access (IIS)
HTTPS (optional)TCP 4430.0.0.0/0, ::/0Secure web access
ICMP (optional)Echo RequestYour IPPing diagnostics

Lock down RDP: Limit to your public IP or a trusted CIDR block. Remove 0.0.0.0/0 in production.

NACLs & Windows Firewall: Ensure they allow your chosen traffic as well.

Connection Methods

RDP (Primary)

  1. EC2 > Connect > RDP client > Download remote desktop file.
  2. Open the .rdp file (or use the Public IP in your RDP client).
  3. Username: Administrator • Password: decrypted value.

SSH (Optional)

Enable the OpenSSH Server feature on Windows, then allow inbound TCP 22 in the security group:

# In an elevated PowerShell on the Windows instance
Add-WindowsCapability -Online -Name OpenSSH.Server~~~~0.0.1.0
Start-Service sshd
Set-Service -Name sshd -StartupType 'Automatic'
# Allow Windows Firewall
New-NetFirewallRule -Name sshd -DisplayName 'OpenSSH Server (sshd)' -Enabled True -Direction Inbound -Protocol TCP -Action Allow -LocalPort 22

Then from your client:

ssh Administrator@<Public-IP>

What You Should See

  • Machine name, current user, Windows version.
  • IP configuration (private & public via NAT), DNS.
  • Connectivity test results (ping/NSLookup).
  • Listening ports and active connections.
  • Directory listing and system information summary.
Tip: To install IIS for a quick HTML landing page, run in PowerShell:
Install-WindowsFeature -name Web-Server -IncludeManagementTools
Place HTML files in C:\inetpub\wwwroot\.
